## 2. 模块业务逻辑（一期相关）

### 2.1 登录注册 `NXone-登录注册`

**入口状态：**

```
未登录 → 登录（手机 | 邮箱 | 密码）| 注册（手机 | 邮箱）
```

**历史稿屏幕（逻辑顺序）：**

| 步骤 | 屏幕名 | 业务要点 |
|------|--------|----------|
| 1 | 登录-手机未输入 / 登录-手机 | 手机号 + 区号；空态与已输入态分离 |
| 2 | 登录-邮箱未输入 / 登录-邮箱 | 邮箱格式校验 |
| 3 | 登录-验证码未输入 / 登录-验证码 | OTP 分格输入；可重发 |
| 4 | 登录-密码登录 | 密码登录备选路径 |
| 5 | 登录-安全验证 | 2FA / 风控二次验证 |
| 6 | 注册-手机 / 注册-邮箱 | 注册与登录分流 |
| 7 | 注册-设置密码 / 注册-重置密码 | 密码强度、确认密码 |
| 8 | 面容识别 | 生物识别（可选能力） |

**CoinLab 一期采纳：**

- 第三方邮箱注册（产品脑图）
- 验证码 + 密码双路径
- 安全验证作为敏感操作前置（非视觉）

**CoinLab 一期不采纳：**

- 历史稿具体表单布局；改用 MASTER 极简表单

---

### 2.2 首页 `Nxone-首页`

**历史信息架构（从 Frame 命名推断）：**

```
状态栏 → 导航栏 → 搜索
→ 金刚区（快捷入口）
→ Banner
→ 行情/榜单区块（多 Tab 列表行）
→ [历史特有] AI 推荐区
→ 底部 Tab
```

**CoinLab 一期映射：**

| 历史能力 | 一期 |
|----------|------|
| 搜索 | 保留入口 |
| 金刚区 | 收敛为 4 图标（充/提/客服/更多） |
| Banner | 单条新手指引（非运营轮播） |
| 行情列表 | 保留（自选/全部） |
| AI 区 | **不做** |
| 启动页 Light/Dark | 二期品牌定稿后再做 |

### 2.3 行情 `NXone-行情`

**屏幕：**

| 屏幕 | 逻辑 |
|------|------|
| 行情-1/2/3 | 列表视图变体（涨跌、排序） |
| 管理 / 管理-编辑 | 自选增删、排序 |
| 搜索-弹出 | 全站交易对搜索 |
| 现货 | 现货行情列表（**一期对标**） |
| 合约 | 合约列表（**非一期**） |

**业务规则：**

- 自选与全部分 Tab
- 支持编辑自选顺序
- 搜索为模态/全屏层，不离开行情模块

---

### 2.4 交易 `NXone-交易`

**现货相关（一期）：**

- 主交易屏：交易对 + K 线区 + 订单簿 + 下单面板
- 买入/卖出切换
- 限价 / 市价（历史还有大量合约委托类型 → **一期不做**）

**历史存在但一期裁剪：**

分时委托 · 合约偏好 · 分仓 · 加仓 · 单向持仓 · 止盈止损 · 追踪委托 · 计划委托 · 下单器全量类型编辑 · 逐仓保证金 …

**下单业务规则（从多版「交易」Frame 归纳）：**

1. 选择方向（买/卖）
2. 选择类型（限价/市价）
3. 输入价格（限价）、数量
4. 展示：可用余额、预估成交额、手续费
5. 提交 → 确认（历史有独立修改委托单流程）
6. 订单管理：当前委托 / 历史委托（在交易页 Tab 或子页）

---

### 2.5 资产与充提 `NXone-资产-充提`

**资产总览：**

| 屏幕 | 逻辑 |
|------|------|
| 资产 / 资产-现货 | 总资产 + 分账户（现货/合约/理财…） |
| 资产详情 | 单币种余额、冻结、可用 |

**充币流程：**

```
选币种 → 选网络/链 → 展示地址/二维码 → 复制/分享
→ [可选] 充币-点编辑（地址簿）
```

**提币流程：**

```
选币种 → 填地址（支持扫一扫）→ 填数量 → 手续费/到账
→ 安全验证 → 提交
→ 状态：转账中 | 成功 | 失败（提币分享-*）
```

**其他：**

| 屏幕 | 一期 |
|------|------|
| 账单 | **是**（流水） |
| 资金划转 | 视是否多账户；一期可简化为单现货账户 |
| 合约 / 理财 / AI-我的策略 | **否** |
| 余额页面展示管控资金入口 / AML | **合规参考**（提示逻辑，非 UI） |

**CoinLab 一期链与币种（产品脑图）：** BSC、TRC20 · USDT、BNB、TRX

---

### 2.6 我的 / 账户 `NXone-我的`

**用户中心：**

- 昵称、头像、简介编辑
- 语言切换、主题模式（历史有；CoinLab 一期可仅语言）

**KYC 状态机（历史为多级）：**

```
未认证 → 上传证件照 → 审核中 → 1级完成
→ 2级（地址照/视频）→ 3级 …
```

**CoinLab 一期（Sumsub 或自审）：**

- 采纳：**状态展示**（未认证 / 审核中 / 已认证）
- 采纳：未完成 KYC **限制提币**（已在原型体现）
- 不采纳：三级 KYC 复杂度的 UI 逐步展开（按实际合规裁剪）

**安全设置（历史稿存在）：**

- 谷歌验证器
- 手势解锁
- 安全验证要求
- 修改登录/支付密码（产品脑图已有）

## 4. 推荐页面流（CoinLab 实现用）

以下为 **逻辑流**，非 UI 线框：

### 4.1 游客

```
首页（欢迎 + 购买指引 Banner + 行情只读）
  → 注册/登录
```

### 4.2 已登录

```
首页（余额 + 快捷 + 行情）
  → 行情详情 → 交易（现货）
  → 资产 → 充币 | 提币 | 账单
  → 我的 → KYC | 安全 | 语言 | 帮助
```

### 4.3 提币（含合规）

```
资产 → 提币 → 填单 → 安全验证
  → [若未 KYC] 拦截 + 引导认证
  → 提交 → 转账中 → 成功/失败
```
